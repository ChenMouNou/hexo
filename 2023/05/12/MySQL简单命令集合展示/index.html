(feihua可以不看)数据库的基本概念

数据库Database简称DB 是指存储在计算机内得，按一定规则进行组织的，可共享得数据集合。

数据库是数据的仓库，与普通文件不同，数据库依靠一定的得数据结构来存储数据。

数据库系统 Database System 简称 DBS 通常是指数据库及其管理软件组成的系统。

数据库管理系统 Database Management System 简称 DBMS 是操作管理数据库的软件，用于建立，使用和维护数据库。DBMS 对数据库进行统一管理和控制，以保证数据库的安全和完整性。用户和管理员可以通过DBMS 访问数据库录中的数据，进行数据库得维护工作。

大部分DBMS 提供数据定义语言：Data Definiton Language 简称 DDL，和数据操作语言（Data Manipulation Language 简称DML，提供给用户用于定义数据库得模式结构与权限约束，实现用户对数据进行插入，修改，查询，删除等操作。

除了使用DBMS 管理，维护数据库，大部分数据库均支持通过数据接口提供各种编程语言连接数据库和读写数据服务。

#Python 语言支持多种数据库访问，通过遵循数据库接口规范PythonDB-API来保证各种Python 数据库接口模块数据库访问方式的规范化。

关系型数据库

关系型数据库是指使用二维表格关系模型组织的数据库库，通过行和列的形式表示和存储数据，数据库由行和列组成的表构成

主流的关系型数据库有Oracle，DB2，MySQL，Server等，每种数据库的语法，功能和特性也各具特色。但一般都支持结构化查询语言（Structured Query Language 简称SQL）对数据库的维护管理和数据的插入，修改，查询，删除操作。

Oracle数据库是由甲骨文公司开发的，在集群技术，高可用性，安全性，系统管理等方面都有较好的性能表现，支持跨平台运行，是目前大型高性能商务数据库的首选。

MySqL是一款开源系关系型数据库系统，具备体积小，速度快，部署成本低等优点，备受中小型企业欢迎。

MySQL

MySQL是一款开源关系型数据库，具有安全，跨平台，高效等特点，还为PHP，Java，Python等多种主流编程语言提供程序编程接口（Application Programming Interface简称API）支持。

公司简介

1995年，David Axmark、Allan Larsson和Michael Monty Widenius在瑞典创建MySQL AB公司，负责MySQL的开发，发布等工作。

2008年被Sun Microsystem公司收购。Sun公司进行推广，优化，Bug修复等工作。

2009年甲骨文公司收购Sun公司并进行大量优化。

基本命令

提醒

MySQL命令不区分大小写

简单日常命令

CREATE Database; 数据库名：创建数据库

DROP database 库名 删除库

SHOW Databases; 查看数据库

USE 库名； 进入数据库

SHOW TABLES; 查看当前数据库的表

show VARIABLES LIKE ‘%CHARACTER%’ 查看数据库服务器和数据库的字符集

show CHARSET 查看mysql支持的数据库字符集

show CREATE DATABASE 库 查看库的字符集格式

插入数据

insert INTO 表名(字段1,字段2,字段3…)

VALUES(值1,值2,值3…) 这两条用来在表中插入数据（中文要加引号，特殊的值也要加如：’ 1999-8-15’和’ chen’这样的）

VALUSE(值), (值), (值), (值) 就可以插入多条值

查找数据

Select 字段名1,字段名2 FROM 表名 WHERE 字段=值(这个字条件可以使用运算符还有‘and’，‘or’等) LMIT 指定返回的记录的数量 ORDER BY 排列方式（可以使用ASC和DESC来选择升降默认为升序）

如：

SELECT * FROM 表 ORDER BY uID DESC LIMIT 5; （把LIMIT放到ORDER后面 不然会报错，*为所有字段）整条命令是返回表的所有字段倒序的五条（就是倒数五条）

SELECT * FROM 表 WHERE uID=1 or uID=5; （查询表的uID1和5）
SELECT * FROM 表 WHERE uID<5 or uName=’wang’; (查询uID小于5的和uName为wang的)

修改表数据

UPDATE 表 SET uName=’chen’,USex=’女’ WHERE uID=1; （将表中uID为1的进行修改，修改为‘chen’，‘女’） 主要使用WHERE来查找，再修改。

删除表数据

DELETE FROM 表 WHERE uName=’long’; （删除表中uName为long的数据）

建表：

CREATE table ‘表名’(

​ ‘字段名’ 数据类型 Default ((就是默认值) AUTO_INCREMENT(设置字段自增) COMMENT（注释） UNIQUE（唯一约束） PRIMARY KEY （主键）

）ENGINE=InnoDB（设置表的引擎为InnoDB） DEFAULT CHARSET=utf-8 设置默认utf-8字符集

如：

CREATE TABLE lun.No1(

uID int(10) PRIMARY KEY auto_increment COMMENT ‘ID’,

uName VARCHAR(15) not null,

USex enum(‘男’,’女’) default ‘女’);

建表时出现报错先修改库的字符集格式,有可能时格式有问题,先来改下格式（没有特殊要求就改）,如果你在navicat里直接右击表数据库就可以编辑。

也可以直接建表的时候就写好表的字符集格式及排列方式

CREATE DATABASE 表名 CHARACTER set utf8 COLLATE utf8_general_ci （红色是字符集和排列方式）

不在navicat的可以输入命令

ALTER DATABASE 库 CHARACTER SET gb2312 COLLATE gb2312_chinese_ci 进行修改。
